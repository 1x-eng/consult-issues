# consult issues

the source for [consult](https://pypi.org/project/getconsult/) is private. this repo is where you file bugs, request features, or ask questions.

## filing an issue

check [existing issues](https://github.com/1x-eng/consult-issues/issues) first. if you're reporting a bug, include `consult --version` and the error output (redact your api keys).

## what consult is

multi-agent consensus cli. n llm agents analyze your problem independently, review each other's work, iterate until they agree, then one synthesis agent merges it into a single recommendation.

```bash
pip install getconsult
```

docs: [getconsult.sysapp.dev](https://getconsult.sysapp.dev)
